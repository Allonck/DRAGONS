addDQ(rc)[source]¶

This primitive will create a numpy array for the data quality of each SCI frame of the input data. This will then have a header created and append to the input using AstroData as a DQ frame. The value of a pixel will be the sum of the following: (0=good, 1=bad pixel (found in bad pixel mask), 2=value is non linear, 4=pixel is saturated)

Parameters:
  • suffix (string) – Value to be post pended onto each input name(s) to create the output name(s).
  • fl_nonlinear – Flag to turn checking for nonlinear pixels on/off
  • fl_saturated (Python boolean (True/False), default is True) – Flag to turn checking for saturated pixels on/off
  • logLevel (int. This value can be set for each primitive individually in a recipe only (ie. not in the parameter file). If no value is specified during the recipe, the value set during the call to reduce or its default (2) will be used.) – Verbosity setting for log messages to the screen.
addVAR(rc)[source]¶

This primitive uses numpy to calculate the variance of each SCI frame in the input files and appends it as a VAR frame using AstroData.

The calculation will follow the formula: variance = (read noise/gain)2 + max(data,0.0)/gain

Parameters:
  • suffix (string) – Value to be post pended onto each input name(s) to create the output name(s).
  • logLevel (int. This value can be set for each primitive individually in a recipe only (ie. not in the parameter file). If no value is specified during the recipe, the value set during the call to reduce or its default (2) will be used.) – Verbosity setting for log messages to the screen.

combine(rc)[source]¶

This primitive will average and combine the SCI extensions of the inputs. It takes all the inputs and creates a list of them and then combines each of their SCI extensions together to create average combination file. New VAR frames are made from these combined SCI frames and the DQ frames are propagated through to the final file.

Parameters:
  • suffix (string) – Value to be post pended onto each input name(s) to create the output name(s).
  • fl_vardq (Python boolean (True/False), OR string ‘AUTO’ to do it automatically if there are VAR and DQ frames in the inputs. NOTE: ‘AUTO’ uses the first input to determine if VAR and DQ frames exist, so, if the first does, then the rest MUST also have them as well.) – Create variance and data quality frames?
  • fl_dqprop (Python boolean (True/False)) – propogate the current DQ values?
  • method (string, options: ‘average’, ‘median’.) – type of combining method to use.
  • logLevel (int. This value can be set for each primitive individually in a recipe only (ie. not in the parameter file). If no value is specified during the recipe, the value set during the call to reduce or its default (2) will be used.) – Verbosity setting for log messages to the screen.

measureIQ(rc)[source]¶

This primitive will detect the sources in the input images and fit both Gaussian and Moffat models to their profiles and calculate the Image Quality and seeing from this.

Parameters:
  • function (string, can be: ‘moffat’,’gauss’ or ‘both’; Default ‘both’) – Function for centroid fitting
  • display (Python boolean (True/False) Default: True) – Flag to turn on displaying the fitting to ds9
  • mosaic (Python boolean (True/False) default: True) – Flag to indicate the images have been mosaic’d (ie only 1 ‘SCI’ extension in images)
  • qa (Python boolean (True/False) default: True) – flag to use a grid of sub-windows for detecting the sources in the image frames, rather than the entire frame all at once.
  • logLevel (int. This value can be set for each primitive individually in a recipe only (ie. not in the parameter file). If no value is specified during the recipe, the value set during the call to reduce or its default (2) will be used.) – Verbosity setting for log messages to the screen.

standardizeStructure(rc)[source]¶

This primitive ensures the MEF structure is ready for further processing, through adding the MDF if necessary and the needed keywords to the headers. First the MEF’s will be checked for the general Gemini structure requirements and then the instrument specific ones if needed. If the data requires a MDF to be attached, use the ‘addMDF’ flag to make this happen (eg. standardizeStructure(addMDF=True)).

Parameters:
  • suffix (string) – Value to be post pended onto each input name(s) to create the output name(s).
  • addMDF (Python boolean (True/False) default: True) – A flag to turn on/off appending the appropriate MDF file to the inputs.

validateData(rc)[source]¶

This primitive will ensure the data is not corrupted or in an odd format that will affect later steps in the reduction process. It will call a function to take care of the general Gemini issues and then one for the instrument specific ones. If there are issues with the data, the flag ‘repair’ can be used to turn on the feature to repair it or not (eg. validateData(repair=True)) (this feature is not coded yet).

Parameters:
  • suffix (string) – Value to be post pended onto each input name(s) to create the output name(s).
  • repair (Python boolean (True/False) default: True) – A flag to turn on/off repairing the data if there is a problem with it. Note: this feature does not work yet.
  • logLevel (int. This value can be set for each primitive individually in a recipe only (ie. not in the parameter file). If no value is specified during the recipe, the value set during the call to reduce or its default (2) will be used.) – Verbosity setting for log messages to the screen.
writeOutputs(rc)[source]¶

A primitive that may be called by a recipe at any stage to write the outputs to disk. If suffix is set during the call to writeOutputs, any previous suffixs will be striped and replaced by the one provided. examples: writeOutputs(suffix= ‘_string’), writeOutputs(prefix= ‘_string’) or if you have a full file name in mind for a SINGLE file being ran through Reduce you may use writeOutputs(outfilename=’name.fits’).

Parameters:
  • strip (Python boolean (True/False) default: False) – Strip the previously suffixed strings off file name?
  • clobber (Python boolean (True/False) default: False) – Write over any previous file with the same name that all ready exists?
  • suffix (string) – Value to be post pended onto each input name(s) to create the output name(s).
  • prefix (string) – Value to be post pended onto each input name(s) to create the output name(s).
  • outfilename (string) – The full filename you wish the file to be written to. Note: this only works if there is ONLY ONE file in the inputs.
  • logLevel (int. This value can be set for each primitive individually in a recipe only (ie. not in the parameter file). If no value is specified during the recipe, the value set during the call to reduce or its default (2) will be used.) – Verbosity setting for log messages to the screen.
