fringeCorrect(rc)[source]¶

This primitive will scale and subtract the fringe frame from the inputs. It utilizes the Python re-written version of cl script girmfringe to do the work.

Parameters:
  • postpend (string) – Value to be post pended onto each input name(s) to create the output name(s).
  • fl_statscale (Boolean) – Scale by statistics rather than exposure time
  • statsec (string of format ‘[EXTNAME,EXTVER][x1:x2,y1:y2]’ default: If CCDSUM = ‘1 1’ :[SCI,2][100:1900,100:4500]’ If CCDSUM = ‘2 2’ : [SCI,2][100:950,100:2250]’) – image section used to determine the scale factor if fl_statsec=True
  • scale (real) – Override auto-scaling if not 0.0
  • logVerbose (int. This value can be set for each primitive individually in a recipe only (ie. not in the parameter file). If no value is specified during the recipe, the value set during the call to reduce or its default (2) will be used.) – Verbosity setting for log messages to the screen.
makeFringeFrame(rc)[source]¶

This primitive will create a single fringe image from all the inputs. It utilizes the CL script gifringe to create the fringe image.

Parameters:
  • postpend (string) – Value to be post pended onto each input name(s) to create the output name(s).
  • fl_vardq (Python boolean (True/False), OR string ‘AUTO’ to do it automatically if there are VAR and DQ frames in the inputs. NOTE: ‘AUTO’ uses the first input to determine if VAR and DQ frames exist, so, if the first does, then the rest MUST also have them as well.) – Create variance and data quality frames?
  • method (string, options: ‘average’, ‘median) – type of combining method to use on the fringe frames.
  • logVerbose (int. This value can be set for each primitive individually in a recipe only (ie. not in the parameter file). If no value is specified during the recipe, the value set during the call to reduce or its default (2) will be used.) – Verbosity setting for log messages to the screen.
